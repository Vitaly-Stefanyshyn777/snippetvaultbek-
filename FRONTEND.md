# Mini Snippet Vault — Frontend Guidance

Фронтенд обов'язково реалізується в іншій репозиторії/директорії. Нижче вимоги, щоб UI працював з API та непорушував acceptance criteria.

## Стек
- Next.js App Router (TypeScript)
- Tailwind CSS
- `fetch` або `axios` для звернень до API
- Клієнтські компоненти (`'use client'`) там, де є форми/інтерактивні елементи

## Інтеграція з API
- `NEXT_PUBLIC_API_URL` (наприклад `http://localhost:4000`). З нього будується ```${NEXT_PUBLIC_API_URL}/api/snippets```.
- Підтримуються: GET `/snippets` (query-параметри `page`, `limit`, `q`, `tag`), POST `/snippets`, PATCH `/snippets/:id`, DELETE `/snippets/:id`.

## Очікуваний UX
1. **Список**: форма створення (title + content + type + tags), пошук по title/content, клікабельні теги для фільтрації, пагінація (page/limit), тегова навігація, стани loading/empty/error.
2. **Деталі/редагування**: окрема сторінка (`/snippets/[id]`) з показом `createdAt`/`updatedAt`, блоком з контентом/тегами та формою редагування з можливістю видалити.
3. **Валідація**: мінімум `required` з адекватними повідомленнями, форма має блокувати надсилання, поки не заповнені title/content.
4. **Стилі**: Tailwind для читабельного, мінімалістичного інтерфейсу без порожніх екранів.

## Стани
- Loading: поки йде fetch, відображати повідомлення.
- Empty: якщо список порожній — підказка.
- Error: при 4xx/5xx відповідях показати повідомлення.

## Запуск
```bash
npm install
npm run dev
```
Також додати `.env.example` з `NEXT_PUBLIC_API_URL`.

## Перевірка
- Створити snippet у формі, переконатися, що він з'явився в списку з тегами.
- Зробити пошук/фільтр і переконатися, що API повертає обмежений набір.
- Перейти на сторінку `/snippets/:id`, змінити, зберегти і видалити. Перевірити, що відповіді 200/204 і UI реагує.
